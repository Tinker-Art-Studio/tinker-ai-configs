Plan: KPI Simplified Round Mode

Goal: Add a "Simplified Round" toggle to Settings so this assessment round only shows and scores Set & Reflect, Leadership Score (if applicable), and Student Attrition — without losing any of the full KPI framework for the future.
App: tinker-kpi — ~/tinker-kpi/js/app.js
Risk: Low — UI filter only, no data deleted, no schema migration, one boolean saved to Firestore
Reversible: Yes — toggle it off to return to full view instantly

What exists today

The KPI app has two separate KPI systems:

SystemWhat it isDefined in
Per-staff KPIsIndividual staff assessments computed from Google Sheets CSVsKPI_DEFINITIONS (line 9)
Studio KPIsBig-picture business metrics (Fulfillment, Operations, etc.)STUDIO_KPI_DEFINITIONS (line 50)

The three KPIs Christie wants for this round are all in KPI_DEFINITIONS:

KPI IDLabelWho it applies toHow computed
set-reflectSet & ReflectAll rolesAuto from Google Sheets CSV
leadership-scoreLeadership KPI Scoreleadership role onlyAuto from leadership KPI form
survey-churnStudent Attritionlead-teacher onlyAuto from parent survey

The function getPersonKPIs(member) (line 2767) builds the list of KPI IDs for each member based on their roles. This is the single place to apply the filter — touching it here means the filter flows automatically through card rendering, the "KPIs met" badge, and the bonus calculation.

Approach

Add a simplifiedRoundMode boolean to the Firestore kpiData document (alongside existing settings like thresholds, semesters, etc.). When true:

  • getPersonKPIs() filters its result to only set-reflect, leadership-score (if applicable to member), and survey-churn (if applicable)
  • The "KPIs met" badge and bonus calculation naturally reflect only these KPIs
  • All other KPI definitions, data, and overrides stay untouched in the background
✅ No data is deleted. No definitions are removed. Studio KPIs are unaffected. Toggle off → full view returns instantly.

Phases

Phase 1 — Settings toggle (UI + Firestore save)

Acceptance criteria: A "Simplified Round Mode" toggle appears in the Settings tab. Turning it on saves simplifiedRoundMode: true to Firestore. Turning it off saves false. The saved value persists across page reloads.

Given: Admin opens Settings tab When: They see "Simplified Round Mode" toggle (off by default) Then: The toggle shows the current saved state from Firestore Given: Admin turns the toggle ON When: They click it Then: updateDoc saves { simplifiedRoundMode: true } to kpiData doc AND the page reflects the change immediately (no full reload needed) Given: Admin turns the toggle OFF When: They click it Then: updateDoc saves { simplifiedRoundMode: false } to kpiData doc
⚠️ Firestore safety: use updateDoc not setDoc. The kpiData doc holds semesters, thresholds, overrides, and other critical data — a full overwrite would destroy it.

Where in code: Add toggle near the Thresholds section in Settings (~line 1724). Save via the same updateDoc pattern used elsewhere in the file.

Phase 2 — Filter getPersonKPIs() when mode is active

Acceptance criteria: When Simplified Round Mode is on, each staff card shows only the KPIs relevant to them from the three allowed: set-reflect, leadership-score, survey-churn. The "X/Y KPIs met" badge reflects only those KPIs. Staff with no applicable KPI from the three (edge case) show a clear "No KPIs this round" message rather than a broken card.

Given: Simplified Round Mode is ON When: I view a lead-teacher's card Then: I see set-reflect + survey-churn only (not shift-reports or any manual KPIs) AND the badge says "X/2 KPIs met" Given: Simplified Round Mode is ON When: I view a leadership-role member's card Then: I see set-reflect + leadership-score only AND the badge says "X/2 KPIs met" Given: Simplified Round Mode is ON When: I view a lead-teacher who also has leadership role Then: I see set-reflect + survey-churn + leadership-score AND the badge says "X/3 KPIs met" Given: Simplified Round Mode is OFF When: I view any staff card Then: All KPIs appear exactly as they did before (no regression)

Where in code: Modify getPersonKPIs(member) (line 2767) to accept a simplified boolean parameter and filter accordingly. Pass the current mode value when calling it (~lines 3080, 3291).

Phase 3 — Visual indicator that simplified mode is active

Acceptance criteria: When Simplified Round Mode is ON, a small banner or pill appears at the top of the Staff KPIs tab so it's obvious the view is filtered. No banner when mode is off.

Given: Simplified Round Mode is ON When: I open the Staff KPIs tab Then: A banner reads "Simplified Round — showing Set & Reflect, Leadership Score, and Student Attrition only" AND a link/button in the banner goes directly to Settings to toggle it off Given: Simplified Round Mode is OFF When: I open the Staff KPIs tab Then: No banner appears

What this does NOT change

  • Studio KPIs tab — unaffected
  • Leadership tab — unaffected
  • Incident Log tab — unaffected
  • Customer Survey tab — unaffected
  • Any KPI definitions, thresholds, or historical data
  • Manual KPI overrides already saved — they persist, just hidden in simplified mode

One question before executing
