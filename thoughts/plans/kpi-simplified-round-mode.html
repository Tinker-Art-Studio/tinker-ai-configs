How it works

Settings gains a new section: "KPIs Active This Round." It shows every KPI from KPI_DEFINITIONS as a labeled checkbox row. Checked = counts this round. Unchecked = hidden from cards and excluded from bonus.

Firestore stores the active list as activeKpiIds: ['set-reflect', 'survey-churn', 'leadership-score'] in the existing kpiData document. Default when the field is absent = all KPIs active (so existing sessions are unaffected until you uncheck something).

✅ Nothing is deleted. Unchecked KPIs still exist in KPI_DEFINITIONS, their data still loads from CSVs, and any saved overrides are preserved. They just don't appear on cards or count toward bonuses.

Phases

Phase 1 — Firestore field + Settings checklist UI

Acceptance criteria: A "KPIs Active This Round" section appears in Settings. Each KPI has a labeled checkbox. Toggling any checkbox immediately saves the updated activeKpiIds array to Firestore via updateDoc. The saved state survives a page reload.

Given: Admin opens Settings tab When: They scroll to "KPIs Active This Round" Then: Every KPI from KPI_DEFINITIONS appears as a labeled checkbox row AND checked state reflects what's saved in Firestore (or all checked if field absent) Given: Admin unchecks "End of Shift Reports" When: They click the checkbox Then: updateDoc({ activeKpiIds: [...remaining] }) fires immediately AND on page reload, "End of Shift Reports" is still unchecked Given: activeKpiIds field does not exist in kpiData yet (existing sessions) When: The app loads Then: All KPIs are treated as active — no regression for existing users
⚠️ Firestore: use updateDoc only — the kpiData doc contains semesters, thresholds, overrides, and other critical data. Never setDoc.

Where: Add after the Thresholds section in Settings (~line 1750). Read data.activeKpiIds; default to all KPI IDs if absent.

Given: activeKpiIds = ['set-reflect', 'survey-churn', 'leadership-score'] When: I view a lead-teacher's card Then: I see set-reflect + survey-churn (not shift-reports or manual KPIs) AND badge says "X/2 KPIs met" AND bonus is calculated from those 2 KPIs only Given: activeKpiIds = ['set-reflect', 'survey-churn', 'leadership-score'] When: I view a leadership-role member Then: I see set-reflect + leadership-score AND badge says "X/2 KPIs met" Given: activeKpiIds = all KPI IDs (or field absent) When: I view any card Then: Output is identical to the app before this change (no regression)
