Plan: Bonus Lock + Accountant Copy + Employee Email Preview

Goal: Three related features for end-of-semester bonus workflow:
1. Lock bonuses per semester — freezes amounts, signals "done"
2. Copy for Accountant — one-click clipboard list of name + amount
3. Email Preview modal — generates formatted bonus email per employee; Christie copies HTML and pastes into Gmail
App: tinker-kpi — ~/tinker-kpi/js/app.js
Risk: Low — no new Firestore collections; lock is a field on existing kpiData doc
Email sending: Copy-paste into Gmail (no backend needed)

Phase 1 — Lock Bonuses

What it does

A "Lock Bonuses" button in the KPI Review tab header. Locking a semester freezes its bonus display and unlocks the Accountant Copy and Email buttons. An admin can unlock to make corrections.

Given: Bonuses are not locked for the current semester When: Admin clicks "Lock Bonuses" Then: A confirmation dialog appears: "Lock Spring 2026 bonuses? You can unlock later if needed." AND on confirm, lockedBonusSemesters array is updated in kpiData via updateDoc AND the button changes to "🔒 Locked — Unlock" with muted styling AND "Copy for Accountant" and "Email All" buttons appear Given: Bonuses are locked When: Admin clicks "Unlock" Then: Semester is removed from lockedBonusSemesters AND buttons revert to pre-lock state
⚠️ Firestore: updateDoc only. Field: lockedBonusSemesters: ['spring-2026', ...] — array of semester IDs added to existing kpiData doc.

Phase 2 — Copy for Accountant

What it does

Appears once bonuses are locked. Copies a plain-text summary to the clipboard.

Given: Bonuses are locked for Spring 2026 When: Admin clicks "Copy for Accountant" Then: Clipboard receives: Spring 2026 KPI Bonuses ──────────────────────────── Kathy Martin $264.96 Negeen Samimi $83.00 Mackenzie Reinhart $31.80 ──────────────────────────── Total $379.76 AND a toast message says "Copied to clipboard"

Only includes members with a non-zero bonus earned. Inactive members included if they have wages entered for the semester.

Phase 3 — Email Preview Modal

What it does

A "Preview Email" button on each staff card (visible when bonuses are locked). Opens a modal showing the fully formatted email — ready to copy and paste into Gmail.

Modal layout

  • Subject line shown at top (auto-filled, copyable separately)
  • Recipient email shown (from member.emails[0])
  • Formatted email preview exactly matching Christie's past emails
  • Editable message area for the personal paragraph (pre-filled with a template)
  • "Copy Email Body" button — copies the full HTML for pasting into Gmail
  • "Copy Subject" button

Grace Pass

Grace passes already exist — you can mark any week as graced in the weekly Set & Reflect view, or override any KPI result to met via the existing KPI override button on each card. When an override marks a KPI as met, the email shows "Grace Pass ✓" in gold automatically.

Not in scope

  • Actually sending emails from the app — Preview & Copy only
  • "Email All" batch button — one at a time (intentional: Christie personalizes each)
